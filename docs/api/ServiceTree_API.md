# 服务树 API 文档

## 概述

服务树模块用于按业务维度组织和管理云资源。核心概念：

- **环境 (Environment)**: 开发/测试/预发/生产等部署环境
- **服务树 (ServiceTree)**: 业务线 → 产品 → 模块的层级结构
- **资源绑定 (Binding)**: 将云资源绑定到指定节点和环境

## 通用说明

### 请求头

| Header       | 必填 | 说明             |
| ------------ | ---- | ---------------- |
| X-Tenant-ID  | 是   | 租户ID           |
| Content-Type | 是   | application/json |

### 响应格式

```json
{
  "code": 0,
  "msg": "success",
  "data": {}
}
```

## 二、服务树节点管理

### 2.1 创建节点

```
POST /api/v1/cam/service-tree/nodes
```

**请求体:**

```json
{
  "uid": "ecommerce.mall.order",
  "name": "订单服务",
  "parent_id": 5,
  "owner": "张三",
  "team": "交易团队",
  "description": "处理订单相关业务",
  "tags": ["核心", "高可用"],
  "order": 1
}
```

**字段说明:**
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| uid | string | 否 | 节点唯一标识，建议用点分格式 |
| name | string | 是 | 节点名称 |
| parent_id | int64 | 否 | 父节点ID，0表示根节点 |
| owner | string | 否 | 负责人 |
| team | string | 否 | 所属团队 |
| description | string | 否 | 描述 |
| tags | []string | 否 | 标签 |
| order | int | 否 | 排序权重 |

### 2.2 获取节点列表

```
GET /api/v1/cam/service-tree/nodes
```

**查询参数:**
| 参数 | 类型 | 说明 |
|------|------|------|
| parent_id | int64 | 父节点ID |
| level | int | 层级 (1=业务线, 2=产品, 3=模块...) |
| name | string | 名称模糊搜索 |
| owner | string | 负责人 |
| team | string | 团队 |
| status | int | 状态 |

**响应示例:**

```json
{
  "code": 0,
  "data": {
    "list": [
      {
        "id": 1,
        "uid": "ecommerce",
```

## 三、资源绑定

### 3.1 绑定资源到节点

```
POST /api/v1/cam/service-tree/nodes/:id/bindings
```

**请求体:**

```json
{
  "env_id": 4,
  "resource_type": "instance",
  "resource_id": 1001
}
```

**字段说明:**
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| env_id | int64 | 是 | 环境ID |
| resource_type | string | 是 | 资源类型: instance/asset |
| resource_id | int64 | 是 | 资源ID |

### 3.2 批量绑定资源

```
POST /api/v1/cam/service-tree/nodes/:id/bindings/batch
```

**请求体:**

```json
{
  "env_id": 4,
  "resource_type": "instance",
  "resource_ids": [1001, 1002, 1003]
}
```

### 3.3 获取节点绑定的资源

```
GET /api/v1/cam/service-tree/nodes/:id/bindings
```

**查询参数:**
| 参数 | 类型 | 说明 |
|------|------|------|
| env_id | int64 | 按环境筛选 |
| resource_type | string | 资源类型 |
| bind_type | string | 绑定方式: manual/rule |

**响应示例:**

```json
{
  "code": 0,
  "data": {
    "list": [
      {
        "id": 1,
        "node_id": 12,
        "env_id": 4,
        "resource_type": "instance",
        "resource_id": 1001,
        "bind_type": "manual",
        "create_time": 1706345600000
      }
    ],
    "total": 1
  }
}
```

### 3.4 解绑资源

```
DELETE /api/v1/cam/service-tree/bindings/:id
```

### 3.5 查询资源所属节点

```
GET /api/v1/cam/service-tree/resources/:type/:id/node
```

## 四、绑定规则

### 4.1 创建规则

```
POST /api/v1/cam/service-tree/rules
```

**请求体:**

```json
{
  "node_id": 12,
  "name": "生产环境订单服务自动归属",
  "priority": 10,
  "conditions": [
    { "field": "tag.env", "operator": "eq", "value": "prod" },
    { "field": "tag.service", "operator": "eq", "value": "order" }
  ],
  "enabled": true,
  "description": "自动将带有指定标签的资源绑定到订单服务"
}
```

**条件操作符:**
| 操作符 | 说明 | 示例 |
|--------|------|------|
| eq | 等于 | `{"field": "tag.env", "operator": "eq", "value": "prod"}` |
| ne | 不等于 | |
| contains | 包含 | |
| regex | 正则匹配 | |
| in | 在列表中 | `{"field": "region", "operator": "in", "value": "cn-hangzhou,cn-shanghai"}` |
| exists | 存在 | |

**可用字段:**

- `name` - 资源名称
- `asset_id` - 资产ID
- `model_uid` - 模型UID
- `attributes.xxx` - 动态属性
- `tag.xxx` - 标签

### 4.2 获取规则列表

```
GET /api/v1/cam/service-tree/rules
```

### 4.3 更新规则

```
PUT /api/v1/cam/service-tree/rules/:id
```

### 4.4 删除规则

```
DELETE /api/v1/cam/service-tree/rules/:id
```

### 4.5 执行规则匹配

```
POST /api/v1/cam/service-tree/rules/execute
```

手动触发规则引擎，对未绑定资源执行自动匹配。

- 支持多选
- 显示资源状态
- 过滤已绑定资源
- 支持搜索

### 5.7 典型用户流程

**流程1: 初始化服务树**

```
1. 进入环境管理页 → 点击"初始化默认环境"
2. 进入服务树页面 → 创建根节点 (业务线)
3. 在业务线下创建产品节点
4. 在产品下创建模块节点
```

**流程2: 手动绑定资源**

```
1. 在服务树中选择目标节点
2. 点击"绑定资源"按钮
3. 选择环境 (如: 生产环境)
4. 选择资源类型 (如: ECS实例)
5. 勾选要绑定的资源
6. 确认绑定
```

**流程3: 配置自动绑定规则**

```
1. 进入规则管理页
2. 点击"新建规则"
3. 选择目标节点
4. 配置匹配条件 (如: tag.service = order)
5. 保存并启用规则
6. 点击"执行规则匹配"触发自动绑定
```

**流程4: 查看资源归属**

```
1. 在服务树中选择节点
2. 右侧面板显示绑定的资源列表
3. 可按环境/类型筛选
4. 查看资源详情或解绑
```
